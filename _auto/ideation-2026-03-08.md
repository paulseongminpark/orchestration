## A. mcp-memory 강화

### A1. 복합 스코어링 (importance + recency + similarity)
- **소스**: @joaomoura, @jamesquint
- **현재**: recall = 3-Layer (타입 태그 + 키워드 부스트 + 다양성 상한)
- **적용**: `final_score = similarity×0.5 + decay_score×0.3 + keyword_boost×0.2`
- **구현**: nodes 테이블에 `importance` 컬럼, promote_node 시 증가

### A2. 5가지 인지 연산
- **소스**: @joaomoura (CrewAI v1.10.1)
- encode(remember) ✅ / consolidate ❌ / recall ✅ / extract(enrichment) 부분적 / forget ❌
- **적용**: consolidate = 중복+유사 노드 병합, forget = half-life 감쇠

### A3. 메모리 충돌 감지
- **소스**: @joaomoura
- "PostgreSQL 이전" → "MySQL 이전" 시 구버전 자동 폐기
- **구현**: embedding 유사도 > 0.85 + 동일 타입 + 시간차 > 7일 → 충돌 후보

### A4. Half-life 망각
- **소스**: @joaomoura
- `decay_score = importance × e^(-λ × days_since_access)`
- access 시 갱신, recall 시 decay 반영
- 2,869 노드 전부 동일 가중치 → 자연 감쇠로 노이즈 감소

### A5. Quirk Store (사용자 수정 → 영구 지식)
- **소스**: @jamesquint 전문
- 구체적 구현: pgvector + pg_textsearch(BM25) + RRF
- 수정 → "quirk" 추출 → embedding 저장 → 새 질문마다 hybrid retrieval → context inject
- **적용**: mcp-memory에 `type: "correction"` 노드, recall 시 항상 top-inject

### A6. Reviewed-item Multiplier
- **소스**: @jamesquint 전문
- 인간 승인 knowledge > 자동 학습 knowledge
- **적용**: promote_node된 노드에 1.5x multiplier in recall scoring

### A7. Self-Scoring Loop
- **소스**: @jamesquint 전문
- 3차원 채점: 구조적 정확성 + 실행 신뢰성 + 컨텍스트 정합성
- context-gap brief: 하위질문별 evidence 체크 → gap 시 targeted retry
- **적용**: ai-synthesizer verify barrier에 3차원 채점 추가

### A8. Knowledge Map (발견 경로 저장)

### H6. Claude 플랫폼 API — Compaction + Tool Search 전체 사양
