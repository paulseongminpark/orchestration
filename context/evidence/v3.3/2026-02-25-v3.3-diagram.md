## 4. 체인 아키텍처 (6개)

```
[구현 체인]
implement → code-reviewer(Opus) → commit-writer(Haiku) → project-linker(Sonnet)

[배포 체인]
pf-deployer(Sonnet) → security-auditor(Sonnet) → 사용자 확인 → push

[추출/검증 체인 v3.3]
Gemini 추출(벌크) + Codex 추출(정밀) → Claude verify barrier(3단계) → 사용

[디스패치 체인]
/dispatch → context-linker(Haiku) → meta-orchestrator(Sonnet) → 팀 활성화

[압축 체인]
compressor 7단계 → orch-doc-writer(Opus, 조건부) → doc-syncer(Haiku)

[세션 전환 체인 v3.3]
verify → sync-all → compressor → context-linker → "새 세션 준비 완료"
```

## 5. 데이터 흐름

```
[컨텍스트 오프로딩 흐름]
사용자 "시스템 파악해"
  → Gemini system-scanner (에이전트/스킬/훅 JSON)
  → Gemini state-scanner (크로스 프로젝트 STATE JSON)
  → Codex extract (git 히스토리 JSON)
  → Claude verify barrier (JSON 검증)
  → Claude 시스템 파악 완료 (~25K 토큰, 직접 읽기 대비 88% 절약)

[tech-review QA 흐름]
/tr-verify
  → git pull (최신 포스트)
  → Gemini news-verifier (팩트체크 JSON) ← 병렬
  → Codex content-qa (포맷 QA JSON)     ← 병렬
  → Claude verify barrier + 종합 판단
  → 수정 필요 시 자동 수정 제안

[병렬 코드 리뷰 흐름]
/cross-review (또는 diff 50줄+ 시 자동)
  → code-reviewer(Opus) ← 메인, 항상
  → Codex review --uncommitted ← 보조, 백그라운드
  → Claude 교차: 일치/Codex만/Claude만/둘다 미발견
  → commit-writer
```
