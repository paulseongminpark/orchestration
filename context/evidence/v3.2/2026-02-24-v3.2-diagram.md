## 6. 작업 흐름 타임라인

```
시간 ──────────────────────────────────────────────────────────►

[세션 시작]
  │
  ├─ SessionStart hook (자동)
  │   ├─ 오늘 로그 출력
  │   ├─ 미커밋 상태
  │   ├─ 미반영 결정 (OVERDUE 마커)
  │   └─ live-context.md 최근 10줄
  │
  ├─ /dispatch (수동, 권장)
  │   ├─ context-linker: 크로스세션 스캔
  │   └─ meta-orchestrator: 팀 추천 → 사용자 확인
  │
  │   또는 /morning (수동)
  │   └─ 통합 대시보드 (Inbox + 미반영 + 미커밋 + TODO)
  │
[작업 중]
  │
  ├─ 구현 ────────────────────────────────────────────►
  │   │
  │   ├─ code-reviewer (자동) ← 구현 완료 감지
  │   ├─ commit-writer (자동) ← 리뷰 통과
  │   ├─ project-linker (자동) ← 커밋 완료
  │   │   └─ live-context.md에 기록
  │   │       └─ 다른 세션에서 자동 인식
  │   │
  │   ├─ (필요시) /dispatch ← 방향 전환
  │   └─ (필요시) /todo add "항목"
  │
[세션 종료]
  │
  ├─ /compressor (수동)
  │   ├─ 1~7단계: 세션 압축
  │   ├─ 8단계: orch-doc-writer (시스템 변경 시)
  │   └─ 9단계: doc-syncer (STATE.md 검증)
  │
  └─ /sync-all (수동)
      └─ 전체 프로젝트 git push + MEMORY 동기화
```

## 7. 서브에이전트 관계

Claude Code에서 "서브에이전트"는 Task 도구로 스폰되는 일시적 프로세스.
에이전트(.md 파일)는 서브에이전트의 행동 정의.

```
메인 Claude (사용자와 대화)
  │
  ├─ Task(subagent_type="general-purpose") → 에이전트.md 기반 행동
  │   예: code-reviewer.md → Opus 서브에이전트 스폰
  │
  ├─ Task(subagent_type="Explore") → 탐색 전용 서브에이전트
  │   예: 코드베이스 검색, 파일 구조 파악
  │
  ├─ Task(subagent_type="Plan") → 설계 전용 서브에이전트
  │   예: 구현 플랜 설계
  │
  └─ TeamCreate → 팀 생성 → Task로 팀원 스폰
      예: build 팀 → code-reviewer + pf-reviewer + commit-writer 스폰
```

팀 스폰 시: 각 멤버가 독립 서브에이전트로 실행. SendMessage로 소통.

## 8. tmux 멀티세션 구조

```
tmux 세션 레이아웃 (예시):
┌──────────────────┬──────────────────┐
│ pane 0:          │ pane 1:          │
│ portfolio        │ monet-lab        │

│ claude code      │ claude code      │

├──────────────────┼──────────────────┤
│ pane 2:          │ pane 3:          │

세션 간 소통: live-context.md (자동)
```
