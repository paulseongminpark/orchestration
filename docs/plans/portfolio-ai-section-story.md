## 포트폴리오 작업 고충 — 전체 맥락

> GPT 대화에서 가져온 내용. 포트폴리오 AI 방법론 섹션에 "왜 이 시스템이 필요했는가"의 맥락으로 활용.

"생각은 많은데, 그것이 구조로 정리되지 않으면 아무 의미가 없다"는 압박이 항상 존재함.

감성 기반 프로젝트(PMCC, 미학, 공간 사고)를 데이터 기반 프로젝트와 동일한 무게로 설득해야 하는 부담.

말로는 설명되지만, 코드·레이아웃·정보 구조로 번역하는 과정에서 계속 막힘.

OpenAI 스타일 UI를 따라가려 할수록 디테일이 끝없이 늘어나는 구조적 피로.

TOC, Scrollspy, 840px 중앙 정렬, 반응형 타이포 등 UI 시스템 정합성 유지가 가장 큰 기술적 난제.

프로젝트 상세 페이지가 "스토리"가 아니라 "판단 구조"로 보이게 만드는 데 어려움.

이미지 배치가 단순 시각 자료가 아니라 논리 증거가 되게 만드는 작업이 계속 반복됨.

Hero–Problem–Structure–Implementation–Decision–Output 흐름을 유지하는 데 지속적인 수정 필요.

About / Operating Principles / Thinking System을 프로젝트들과 논리적으로 연결하는 데 반복 재설계 발생.

Work 탭 → 상세 연결 모드 → 브라우저 back/forward 동기화 구현 과정에서 해시·라우팅 충돌 문제 발생.

W1~W5 단계별 구현 중 매번 "이게 최종 구조인가?"라는 구조 불안정성 발생.

md → JSON → UI 렌더링 구조를 어떻게 가져갈지 계속 고민.

콘텐츠는 완성됐는데, 표현 구조가 완성되지 않은 상태가 반복됨.

감각·미학 기반 사고를 데이터·지표 기반 사고와 같은 설득력으로 표현해야 하는 긴장감.

Empty House CPS의 온톨로지 레이어를 시각적으로 설명하는 데 많은 반복 수정.

Skin Diary AI의 bounding box, heatmap, ERD 등을 논리 흐름에 맞게 배치하는 데 구조 고민 지속.

PMCC를 단순 커뮤니티가 아니라 "조건 설계 실험"으로 보이게 만드는 프레이밍 작업 반복.

UI 톤을 과장 없이 "보고서 톤"으로 유지하려는 자제와 마케팅식 표현 사이의 균형 고민.

전체 사이트가 한 사람의 사고 체계처럼 읽히게 만들고 싶은데, 페이지 단위로 쪼개지며 통합감이 깨지는 문제.

Work 카드가 예쁘게 보이는 것보다, 구조적으로 일관되게 보이는 것이 더 어렵다는 점 체감.

OpenAI 톤을 참고하지만, 그대로 베끼면 정체성이 사라지는 딜레마.

Hero 영상과 이미지 밀도가 높아질수록 페이지 무게감과 속도 문제 발생.

폴더 구조(public/images/projects/…) 정리 자체도 계속 리팩토링 대상이 됨.

GitHub Pages / context-repo / portfolio_ui_test_v2 분리 운영에서 경로 혼란 발생.

SoT(Obsidian)와 실제 배포 구조 사이에서 싱크 관리 피로 누적.

Claude는 실행, GPT는 사고 확장이라는 역할 분리 체계를 유지하는 데 지속적 조정 필요.

log / tracker / packet 체계를 정착시키는 과정에서 형식 반복 수정.

"생각은 휘발, 기록은 남는다"는 원칙을 실제 코드 레벨에서 지키는 게 쉽지 않음.

UI 수정과 구조 수정이 동시에 일어나며 스코프가 계속 확장되는 현상.

매일 "today"를 호출해도 구조가 완전히 고정되지 않았다는 느낌이 남음.

기술 문제(WSL, Warp, keybinding, GitHub Desktop 등)가 포트폴리오 본질 작업을 방해하는 순간들.

이미지 배치 하나에도 논리 구조가 어긋나면 다시 전체 흐름을 건드려야 하는 구조적 피로.

콘텐츠 밀도는 높은데, 시각적 여백 설계가 계속 부족하게 느껴짐.

80% 줌 체감 배율, clamp 타이포, 간격 단위까지 미세 조정 반복.

상세 페이지 글자 크기와 글로벌 탭 글자 크기 통일 작업에서 스타일 체계 충돌.

Work 3개 프로젝트가 서로 다른 분야인데도 한 사고 체계로 묶어야 하는 부담.

스스로를 "시스템 설계자"로 보여주고 싶은데, 화면은 여전히 UI 프로젝트처럼 보일까 봐 고민.

프로젝트 설명이 아니라 "판단 기준"을 보여주고 싶은 욕구가 계속 기준을 높임.

결과물보다 구조를 증명하고 싶어서 오히려 난이도가 상승.

완성에 가까워질수록 "이건 아직 아니다"라는 자기 검열이 강해짐.

## 프로젝트 폴더 구조 한 줄 요약

- **01_orchestration** — AI 다중 에이전트 협업을 위한 STATE 기반 컨텍스트 공유 시스템
- **02_portfolio** — OpenAI 스타일 UI 기반의 사고 구조 중심 포트폴리오 웹사이트
- **PMCC** — 조건 설계를 통해 관계 밀도를 실험한 커뮤니티 구조 프로젝트
- **Skin Diary AI** — 이미지 분석 + 날씨 데이터 기반 개인화 스킨 추천 시스템
- **Empty House CPS** — 온톨로지 기반 빈집 전환 가능성 점수화 및 개입 우선순위 시스템
- **content/*.md** — 사고를 UI로 번역하기 위한 판단 단위 텍스트 원본
- **public/images/projects/** — 논리 구조를 증명하기 위한 시각 증거 레이어
- **STATE.md** — 현재 진행 상황을 모든 AI가 공유하는 단일 진실 소스
- **LOGS / EVENTS** — 의사결정의 기록과 히스토리 추적 구조
- **CLAUDE.md 계층** — 실행 범위 통제 및 토큰 최적화를 위한 제어 레이어
