## 3. 그래서 뭘 해야 돼?

### 80개 중 진짜 중요한 건 9개

30개 실행 항목으로 정리했는데, 그 중에서도:

**게임 체인저 3개** (이것 없으면 나머지가 의미 약함):
1. **defer_loading** — MCP 도구 55K 토큰 → 8K. 작업 공간 51% 증가.
   쉽게 말하면: 방 안에 쓸데없는 가구가 반을 차지하고 있었는데, 필요할 때만 꺼내는 거.

2. **복합 스코어링** — recall이 "비슷한 것"만 찾던 걸 → "비슷하고 + 최근이고 + 중요한 것" 순으로.
   쉽게 말하면: 구글 검색이 관련도만 보던 걸 → 관련도 + 최신성 + 신뢰도로 바꾸는 거.

3. **Learn 단계 + TASK_CONTRACT** — 매 세션이 끝날 때 "뭘 배웠나" 자동 추출.
   쉽게 말하면: 일기를 안 쓰면 한 달 후 뭘 했는지 모르는 것과 같음. 자동 일기 시스템.

**포스 멀티플라이어 6개** (게임 체인저 위에 얹으면 곱):
4. Correction 노드 타입 — 틀렸을 때 바로잡은 거 영구 기록
5. governance-audit hook — 위험 행동 실시간 감지
6. MEMORY.md 분리 — 토큰 추가 절감
7. 스킬 TRIGGER 패턴 — 스킬 발동 정확도
8. Memory-Merger — 기억이 성숙하면 자동으로 규칙이 됨
9. Wave DAG — 병렬 에이전트 실행

## 4. 어떤 순서로?

**순서가 중요한 이유**: 1번의 결과가 2번을 더 효과적으로 만들기 때문.

```
Phase 1: 토큰 해방 (3시간)
  → 작업 공간 51% 증가. 이후 모든 작업이 더 여유로워짐.

Phase 2: Recall 혁명 (9시간)
  → 넓어진 공간에서 더 좋은 기억을 꺼냄. NDCG 0.46→0.65.

Phase 3: 학습 루프 (7시간)
  → 좋은 기억 위에 매 세션 학습이 쌓임. 시스템이 성장.

Phase 4: 안전 자율 (3.5시간)
  → 성장하는 시스템을 안전하게 자율 운영.

Phase 5: 확장 (선택)
  → 나머지 nice-to-have들.
```

**핵심**: Phase 1→2→3→4가 **복리**다. 1번이 2번을 증폭하고, 2번이 3번을 증폭한다.
하지만 기술적으로는 Phase 1, 2, 4는 병렬 가능. 동시에 시작할 수 있다.

## 5. 4개 시너지 Combo

80개 인사이트가 흩어져 있으면 1+1=2인데, 묶으면 1+1=5가 되는 조합:

**Combo 1: 토큰 해방**
defer_loading + MEMORY.md 분리 + 캐시 순서
→ 같이 해야 "안정된 prefix → 높은 캐시 히트 → 비용+토큰 동시 절감"

**Combo 2: Recall 혁명**
복합 스코어링 + Half-life + Correction + Multiplier
→ 같이 해야 "1차원 → 4차원 recall". 하나만 하면 부분 효과.

**Combo 3: 학습 루프**
TASK_CONTRACT + Learn 단계 + lessons.md + Memory-Merger
→ "계약→실행→학습→축적→승격" 파이프라인. 빠지면 끊김.

**Combo 4: 거버넌스 스택**
정책(allow/deny/review) + 코드 강제 + 실시간 감시
→ 3중 방어. 하나만으로는 뚫림.

## 6. 비유로 정리

지금 시스템 = **100평 아파트인데 가구가 51평을 차지하고 있고, 서랍에서 물건 찾는 데 3분 걸리고, 뭘 했는지 일기를 안 쓰고, 현관문 열쇠가 항상 꽂혀있는 상태.**

Phase 1 = 안 쓰는 가구 치우기 (51평 → 100평 사용)
Phase 2 = 서랍 정리 (3분 → 30초)
Phase 3 = 일기 쓰기 (매일 뭘 배웠는지 기록)
Phase 4 = 현관문 잠금 장치 (열쇠 꽂아두지 않기)

**22.5시간 투자 → 매 세션이 질적으로 다른 수준의 시스템.**
